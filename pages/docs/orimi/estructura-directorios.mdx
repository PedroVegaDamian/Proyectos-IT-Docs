## src

Es el directorio principal de la aplicación.

### assets

Es el directorio donde se guandan los recursos.

### components

Es el directorio donde se crean los componentes reutilizables que se utilizan en diferentes partes de la aplicación.

### firebase

Es el directorio de configuración de Firebase.

### hooks

Es el directorio donde se crean los hooks.

### models

Es el directorio donde se crean los modelos de datos, como los types e interfaces.

### layouts

Es el directorio donde se crean los layouts.

### pages

Es el directorio donde se crean las páginas que se relaciona con las rutas de la aplicación.

### router

Es el directorio donde se configuran las rutas.

### services

Es el directorio donde se crean los servicios que interactuan con firebase.

### store

Es el directorio donde se maneja los estados globales.

### utils

Es el directorio donde se crean los métodos y funciones auxiliares.

### PascalCase

Ejemplo: `InputForm`, `BaseButton`, `UserResponse`

Se utilizará esta forma de nombrar a los componentes, interfaces y tipos.

### CamelCase

Ejemplo: `handleClick`, `firstName`, `errorMessage`

Se utilizará esta forma de nombrar a las funciones y variables.

### SnakeUpperCase

Ejemplo: `PI`, `API_URL`, `MAX_AGE`

Se utilizará esta forma de nombrar a las constantes.
